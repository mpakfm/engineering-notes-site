# Модель контента

## Типы материалов

### Заметка

Короткий практический материал: наблюдение, вывод, небольшой разбор ситуации.

Поля:

- title;
- slug;
- date;
- summary;
- tags;
- readingTime;
- body.

### Разбор

Более длинный материал по конкретной проблеме.

Рекомендуемая структура:

```markdown
# Название

## Контекст

## Проблема

## Почему простое решение не работает

## Рабочий компромисс

## Что можно улучшить

## Вывод
```

### Чек-лист

Прикладной список для работы.

Поля:

- title;
- slug;
- summary;
- items;
- tags;
- updatedAt.

## Первые материалы для запуска

### С чего начать

1. Большой Bitrix и Git: почему все сложно.
2. Почему маленькая правка редко бывает маленькой.
3. Чек-лист перед интеграционной задачей.
4. Как я оцениваю задачи и закладываю риски.

### Свежие заметки

1. Контент на проде и Git: рабочий компромисс.
2. Почему конфиги нельзя править в чате.
3. Что проверить перед релизом Bitrix-сайта.
4. Как писать ТЗ так, чтобы команды поняли одинаково.
