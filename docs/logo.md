# Обсуждение design/brand-reference-header.png

Вот это уже почти то, что надо.

Я бы сказал: направление выбрано правильно, и этот вариант уже можно использовать как бренд-референс для шапки, но сам знак mp/ я бы еще чуть-чуть дополировал перед финальной вставкой.

*Что хорошо*

1. mp/ работает лучше, чем mp

Slash сразу добавляет технический характер:

```
mp/
```

Это читается как:

кусок пути;
терминальный/файловый мотив;
инженерный маркер;
неформальный интернет-ник.

Это хорошо совпадает с mpakfm.ru и “рабочими заметками инженера”.

2. Header lockup удачный

Связка:

```
[mp/] mpakfm
      рабочие заметки инженера
```

— прям правильная.

Она:

 - объясняет домен;
 - не выглядит как резюме;
 - не делает сайт “корпоративным”;
 - сохраняет личность;
 - хорошо подходит для desktop и mobile.

 3. Цвета нормальные

Muted green + graphite + warm off-white — в тему сайта.

Не глянцево, не маркетингово, не SaaS. Получилось спокойно.

4. Mobile header понятный

На мобильном:

```
[mp/] mpakfm        search burger
```

выглядит хорошо. Даже можно убрать подпись “рабочие заметки инженера” на совсем узких экранах, а оставить только mpakfm.

## Что я бы поправил

1. Знак в квадрате чуть великоват и тяжеловат

Внутри 40×40 иконки буквы занимают много места. В реальной шапке это может выглядеть слегка массивно.

Я бы попросил:

уменьшить mp/ внутри рамки на 8–12%;
сделать рамку тоньше;
добавить чуть больше воздуха;
проверить размер 32×32.
2. Рамка слишком “логотипная”

Скругленный квадрат нормальный, но он сейчас чуть напоминает app icon. Для сайта можно сделать мягче:

либо очень тонкая рамка;
либо почти прозрачный warm background;
либо знак без рамки в desktop, а рамку оставить только для favicon.

Мне нравится компромисс:

в шапке: маленький знак mp/ в тонкой мягкой рамке;
favicon: плотнее, в квадрате;
footer/документы: можно просто текст mp/.

3. В desktop header не нужен burger

На mockup desktop показан burger справа вместе с меню. В реальной desktop-шапке лучше:

```
[mp/] mpakfm       Заметки  Разборы  Чек-листы  Темы  Обо мне
```


А burger только на mobile.

4. В wordmark mpakfm можно сделать чуть спокойнее

Сейчас слово mpakfm довольно крупное и жирное. В шапке сайта оно может перетянуть внимание.

Я бы сделал:

 - mpakfm — medium, не bold;
 - подпись — маленькая, muted green/gray;
 - общий размер шапки компактный.

*Мой финальный выбор*

# По файлу design/web-reference-logo.svg

Я бы сделал знак не через текстовый шрифт, а как простую SVG-монолинию, чтобы он не зависел от установленного шрифта. design/web-reference-logo.svg — рабочий первый вариант mp/ в духе шестого варианта: плоский, спокойный, технический, с мягкой рамкой.

Но я бы для шапки использовал не только иконку, а связку:

```
[mp/] mpakfm
      рабочие заметки инженера
```

Для этого лучше сделать SVG только для маленького знака, а mpakfm и подпись оставить HTML-текстом. Так шапка будет доступнее, гибче и лучше адаптироваться под мобильный.

Например, в BaseLayout.astro:

```
<a href="/" class="brand" aria-label="mpakfm — рабочие заметки инженера">
  <img src="/images/logo-mark.svg" alt="" class="brand-mark" width="40" height="40" />
  <span class="brand-text">
    <span class="brand-name">mpakfm</span>
    <span class="brand-subtitle">рабочие заметки инженера</span>
  </span>
</a>
```

CSS:

```
.brand {
  display: inline-flex;
  align-items: center;
  gap: 0.75rem;
  color: var(--text);
  text-decoration: none;
}

.brand-mark {
  width: 40px;
  height: 40px;
  flex: 0 0 auto;
}

.brand-text {
  display: flex;
  flex-direction: column;
  line-height: 1.1;
}

.brand-name {
  font-size: 1.25rem;
  font-weight: 700;
  letter-spacing: -0.03em;
}

.brand-subtitle {
  margin-top: 0.2rem;

  color: var(--muted);
}

@media (max-width: 420px) {

    width: 36px;
    height: 36px;
  }

  .brand-name {
    font-size: 1.1rem;
  }

  .brand-subtitle {
    font-size: 0.74rem;
  }
}
```

Я бы назвал файл так:

public/images/logo-mark.svg

На 16×16 вся внутренняя детализация m может стать мелковатой, поэтому для favicon я бы позже сделал отдельный favicon.svg с более простым mp/ через текст или упрощенные линии.
