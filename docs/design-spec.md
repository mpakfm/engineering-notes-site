# Дизайн-спецификация

## Общее направление

Светлый личный сайт в стиле инженерного блокнота и практической базы знаний.

Характер:

- спокойный;
- взрослый;
- структурный;
- без маркетингового пафоса;
- с ощущением рабочей тетради, но без чрезмерной декоративности.

Сайт должен выглядеть как место, где инженер аккуратно складывает практический опыт: заметки, разборы, чек-листы, выводы после рабочих задач.

## Основной визуальный референс

Файлы:

```text
/design/descktop-reference.png
/design/mobile-reference.png
/design/brand-reference-header.png
```

Использовать как референсы по настроению, структуре, палитре, типографике и композиции. Desktop и mobile должны быть в одном стиле, но с разной композицией под размер экрана. Не нужно копировать каждый пиксель и весь сгенерированный текст буквально.

`design/homepage-mockup-v1.svg` и `design/homepage-mockup-v1.prompt.md` считать ранними вспомогательными материалами.

## Бренд

Основной знак:

```text
mp/
```

Wordmark:

```text
mpakfm
```

Подпись:

```text
рабочие заметки инженера
```

Логотип должен выглядеть как часть интерфейса инженерного блокнота: минимально, аккуратно, без 3D, свечения, объемных теней и коммерческого брендинга. `design/brand-reference-header.png` — основной референс по шапке и поведению бренда в desktop/mobile.

## Палитра

Основная палитра:

- фон: теплый белый / светлый бежевый;
- карточки: белый с легкой теплой тенью;
- основной акцент: приглушенный зеленый;
- вторичные акценты: спокойный синий, мягкий оранжевый;
- текст: темно-серый / почти черный;
- вторичный текст: серый.

Важно не перегрузить сайт цветами. Зеленый — главный акцент. Остальные цвета использовать дозированно.

Базовые цвета из brand-reference:

- muted green: `#6F8F7B`;
- dark graphite: `#2F2F2F`;
- warm off-white: `#F6F4EF`.

## Типографика

Желательно сочетание:

- выразительный serif или humanist-шрифт для крупного заголовка;
- хорошо читаемый sans-serif для интерфейса, карточек, списков и текста.

Главная задача — читаемость длинных материалов.

## Структура главной страницы

### Шапка

Слева:

- логомарка `mp/`;
- wordmark: `mpakfm`;
- подпись: `рабочие заметки инженера`.

Навигация:

- Заметки;
- Разборы;
- Чек-листы;
- Темы;
- Обо мне.

Иконку поиска можно оставить только если поиск реализован. Переключатель темы убрать из MVP, если темная тема не готова.

### Hero-блок

Заголовок:

```text
Рабочие заметки инженера
```

Подзаголовок:

```text
Практические заметки о веб-разработке, Bitrix, интеграциях, релизах, оценке задач и процессах в команде.
```

Ключевая фраза:

```text
Не идеально, но понятно и работает.
```

Визуальная метафора:

- блокнот / сетка / рабочая тетрадь;
- простая архитектурная схема;
- стикер с коротким псевдокодом или принципом.

Декор должен быть легким. На мобильных спираль блокнота и крупные декоративные элементы можно убирать.

### Блок «С чего начать»

Это вручную выбранные материалы для первого знакомства.

Пример карточек:

1. `Большой Bitrix и Git: почему все сложно`
2. `Почему маленькая правка редко бывает маленькой`
3. `Чек-лист перед интеграционной задачей`
4. `Как я оцениваю задачи и закладываю риски`

### Свежие заметки

Список последних материалов.

У каждой записи:

- иконка или маленькая цветная плашка;
- заголовок;
- краткое описание;
- теги;
- дата;
- время чтения.

### Правая колонка на desktop

Блоки:

1. `Коротко обо мне`
2. `Избранные чек-листы`
3. `Темы`

На мобильных переносить эти блоки вниз после основного контента.

### Footer

Оставить простую фразу:

```text
Документируй то, что узнал сегодня, чтобы завтра не разбираться заново.
```

## Адаптивность

Mobile-first обязателен.

На мобильном порядок блоков:

1. шапка;
2. hero;
3. «С чего начать»;
4. свежие заметки;
5. избранные чек-листы;
6. темы;
7. коротко обо мне;
8. footer.

## Что не делать в MVP

- Не делать тяжелую главную картинку на 1–2 МБ.
- Не оставлять неработающие кнопки поиска/темы.
- Не использовать фальшивые счетчики тем, пока материалов мало.
- Не превращать сайт в резюме или лендинг услуг.
- Не перегружать декоративной «тетрадностью».
