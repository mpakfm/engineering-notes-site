# Prompt / design brief for Image2

Создать clean modern website homepage mockup для личного сайта «Рабочие заметки инженера» Сергея Фомина.

## Общий стиль

Светлый, теплый, минималистичный сайт в стиле инженерного блокнота и практической базы знаний. Не портфолио, не лендинг услуг, не стартаперский SaaS, а спокойный публичный инженерный блокнот.

Настроение: взрослое, структурное, уверенное, без маркетингового пафоса. Должно быть ощущение рабочей тетради: сетка, блокнот, простые схемы, стикеры, аккуратные карточки.

## Палитра

- фон: warm off-white / light beige;
- карточки: white / soft warm white;
- основной акцент: muted green;
- вторичные акценты: muted blue, soft orange;
- текст: near black / dark graphite;
- вторичный текст: gray.

Цвета приглушенные, без кислотности. Зеленый — основной акцент.

## Header

Слева компактный логотип с инициалами `SF`, рядом:

- Сергей Фомин;
- Рабочие заметки инженера.

Навигация:

- Главная;
- Заметки;
- Разборы;
- Чек-листы;
- Темы;
- Обо мне.

Справа иконка поиска. Переключатель темы можно не делать или сделать очень нейтрально.

## Hero

Большая левая карточка как лист инженерного блокнота на сетке.

Текст:

- small label: `инженерный блокнот`;
- headline: `Рабочие заметки инженера`;
- subheadline: `Практические заметки о веб-разработке, Bitrix, интеграциях, релизах, оценке задач и процессах в команде.`;
- три коротких преимущества: `Из практики`, `Проверено`, `Понятно`;
- рукописная фраза: `Не идеально, но понятно и работает.`

Справа в hero простая hand-drawn архитектурная схема:

```text
Bitrix → API → 1C
          ↓
          DB
```

На схеме может быть небольшой sticky note с псевдокодом:

```text
def solve(problem):
    understand(problem)
    simplify()
    document()
```

## Main content

Ниже hero двухколоночная структура.

Левая основная колонка:

1. Блок `С чего начать` — 4 карточки:
   - Большой Bitrix и Git: почему все сложно;
   - Почему маленькая правка редко бывает маленькой;
   - Чек-лист перед интеграционной задачей;
   - Как я оцениваю задачи и закладываю риски.

2. Блок `Свежие заметки` — компактный список последних материалов:
   - Контент на проде и Git: рабочий компромисс;
   - Почему маленькая правка редко бывает маленькой;
   - Чек-лист перед интеграционной задачей;
   - Почему конфиги нельзя править в чате.

Правая колонка:

1. `Коротко обо мне` — без фото, вместо фото круглый аватар с `SF`.
2. `Избранные чек-листы`.
3. `Темы` — без фальшивых счетчиков, просто теги.

## Footer

Фраза:

`Документируй то, что узнал сегодня, чтобы завтра не разбираться заново.`

## Важные ограничения

- Не делать слишком декоративный скетчбук.
- Не делать темный киберпанк.
- Не делать SaaS/agency landing.
- Не перегружать цветами.
- Не использовать фальшивые счетчики, если материалов мало.
- Mobile-first, правую колонку на мобильных переносить вниз.
