# Рабочие заметки инженера

Личный сайт Сергея Фомина: светлый инженерный блокнот для заметок, разборов и чек-листов о веб-разработке, Bitrix, интеграциях, релизах, оценке задач и процессах в команде.

## Суть проекта

Это не портфолио и не маркетинговый блог. Сайт должен восприниматься как публичная инженерная тетрадь: спокойно, практично, структурно, без пафоса.

Главная идея:

> Не идеально, но понятно и работает.

## Что лежит в репозитории

- `design/homepage-mockup-v1.svg` — векторный визуальный референс главной страницы.
- `design/homepage-mockup-v1.prompt.md` — текстовый бриф/промпт для повторной генерации или доработки макета.
- `docs/design-spec.md` — описание дизайна, структуры и требований к верстке.
- `docs/content-model.md` — черновая модель контента: заметки, разборы, чек-листы, темы.
- `content/notes/` — будущие заметки.
- `content/checklists/` — будущие чек-листы.

## Первый MVP

Для первого запуска достаточно:

1. Главная страница.
2. Список заметок.
3. Страница одной заметки.
4. Раздел чек-листов.
5. Страница «Обо мне».

На старте можно не делать поиск, темную тему, подписку и сложную CMS.

## Примечание по макету

Исходный PNG-макет был сгенерирован в чате. GitHub-коннектор в текущем режиме пишет текстовые UTF-8 файлы, поэтому в репозиторий добавлен SVG-референс и подробный промпт. Для постановки задачи на верстку этого достаточно: SVG открывается прямо в GitHub и браузере, а `docs/design-spec.md` фиксирует требования.
