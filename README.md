# Рабочие заметки инженера

Личный сайт Сергея Фомина: светлый инженерный блокнот для заметок, разборов и чек-листов о веб-разработке, Bitrix, интеграциях, релизах, оценке задач и процессах в команде.

## Суть проекта

Это не портфолио и не маркетинговый блог. Сайт должен восприниматься как публичная инженерная тетрадь: спокойно, практично, структурно, без пафоса.

Главная идея:

> Не идеально, но понятно и работает.

## Что лежит в репозитории

- `design/homepage-mockup-v1.svg` — векторный визуальный референс главной страницы.
- `design/homepage-mockup-v1.prompt.md` — текстовый бриф/промпт для повторной генерации или доработки макета.
- мобильный PNG-референс в `design/` — референс мобильной версии главной страницы, загруженный отдельно через GitHub.
- `docs/design-spec.md` — описание дизайна, структуры и требований к верстке.
- `docs/content-model.md` — черновая модель контента: заметки, разборы, чек-листы, темы.
- `content/notes/` — будущие заметки.
- `content/checklists/` — будущие чек-листы.

## Первый MVP

Для первого запуска достаточно:

1. Главная страница.
2. Список заметок.
3. Страница одной заметки.
4. Раздел чек-листов.
5. Страница «Обо мне».

На старте можно не делать поиск, темную тему, подписку и сложную CMS.

## Примечание по desktop-макету

Исходный PNG-макет desktop-версии был сгенерирован в чате. GitHub-коннектор в текущем режиме пишет текстовые UTF-8 файлы, поэтому дополнительно добавлен SVG-референс и подробный промпт. Для постановки задачи на верстку использовать:

- оригинальный PNG, если он добавлен в `design/` вручную;
- `design/homepage-mockup-v1.svg` как векторный референс;
- `docs/design-spec.md` как фиксацию требований.

## Комментарии по мобильному референсу для верстки

Мобильный макет считать основным референсом для адаптива. Он хорошо сохраняет характер desktop-версии и при этом не выглядит как просто «сжатый desktop». Это должна быть полноценная mobile-first версия того же инженерного блокнота.

### Что важно сохранить

- Одноколоночную структуру.
- Компактную шапку с логотипом `SF`, именем, подписью, поиском и burger menu.
- Hero-блок в стиле блокнота: теплый фон, тонкая сетка, заголовок, схема, sticky note и фраза `Не идеально, но понятно и работает`.
- Порядок блоков:
  1. Header.
  2. Hero.
  3. `С чего начать`.
  4. `Свежие заметки`.
  5. `Избранные чек-листы`.
  6. `Темы`.
  7. `Коротко обо мне`.
  8. Footer.
- Теплую светлую палитру.
- Зеленый основной акцент.
- Мягкие карточки с аккуратными скруглениями и легкими тенями.
- Хорошую читаемость текста.

### Что адаптировать, а не копировать слепо

Hero в мобильном референсе удачный, но в реальной верстке его нужно проверить на ширинах 360–390 px. Если схема, sticky note и текст начинают тесниться, декоративную часть можно упростить: схема может быть ниже текста, sticky note можно уменьшить или скрыть на совсем узких экранах.

В блоке `Свежие заметки` метаданные лучше не держать отдельной правой колонкой. На мобильном стабильнее и читабельнее такой порядок:

```text
Заголовок заметки
Краткое описание
Bitrix · Git · Процессы
12 мая 2024 · 12 мин чтения
```

То есть теги, дату и время чтения лучше переносить под описание.

### UX-требования для mobile

- Не показывать полное desktop-меню; использовать burger menu.
- Не делать неработающие элементы: если поиска пока нет, иконку можно убрать или оставить как декоративный pending-элемент только осознанно.
- Не мельчить текст: основной текст около 15–16 px, описания карточек 14–15 px, заголовки карточек 15–17 px, hero headline около 34–40 px.
- Не перегружать декоративной «тетрадностью»: сетка должна быть очень слабой, без тяжелой спирали блокнота на mobile.
- Все карточки должны выглядеть нажимаемыми.
- Между крупными блоками нужен воздух, чтобы страница не ощущалась как плотная админка.
- Footer-фразу оставить спокойной, без чрезмерной цитатной пафосности.

### Итог по mobile

Desktop-концепт и mobile-концепт можно считать готовыми для MVP. В верстке важно сохранить единую визуальную идентичность: светлый инженерный блокнот, практичность, структурность, мягкие карточки и отсутствие маркетингового пафоса.
