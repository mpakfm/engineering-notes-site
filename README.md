# Рабочие заметки инженера

Личный сайт Сергея Фомина: светлый инженерный блокнот для заметок, разборов и чек-листов о веб-разработке, Bitrix, интеграциях, релизах, оценке задач и процессах в команде.

## Суть проекта

Это не портфолио и не маркетинговый блог. Сайт должен восприниматься как публичная инженерная тетрадь: спокойно, практично, структурно, без пафоса.

Главная идея:

> Не идеально, но понятно и работает.

## Примечание по референсам

Для постановки задачи на верстку использовать:

- `design/descktop-reference.png` как основной desktop-референс;
- `design/mobile-reference.png` как основной mobile-референс;
- `design/brand-reference-header.png` как источник правды по бренду `mp/`, логотипу и шапке;
- `docs/design-spec.md` как фиксацию требований.

`design/homepage-mockup-v1.svg` и `design/homepage-mockup-v1.prompt.md` оставить как ранние вспомогательные материалы.

## Комментарии по мобильному референсу для верстки

Мобильный макет считать основным референсом для адаптива. Он сохраняет характер desktop-версии и выглядит как полноценная mobile-first версия инженерного блокнота.

## Что важно сохранить на mobile

- Одноколоночную структуру.
- Компактную шапку с логотипом `mp/`, wordmark `mpakfm`, подписью и burger menu.
- Hero-блок в стиле блокнота: теплый фон, тонкая сетка, заголовок и ключевая фраза.
- Порядок блоков:
  1. Header
  2. Hero
  3. С чего начать
  4. Свежие заметки
  5. Избранные чек-листы
  6. Темы
  7. Коротко обо мне
  8. Footer
- Теплую светлую палитру, muted green акцент и мягкие карточки.

## Что адаптировать, а не копировать слепо

Hero нужно проверять на ширинах 360–390 px: при тесноте упрощать декоративные элементы. В карточках заметок метаданные должны идти под описанием (теги, дата, время чтения), а не правой колонкой.

## UX-требования для mobile

- Не показывать desktop-меню целиком, использовать burger menu.
- Не добавлять неработающие элементы. Иконку поиска из референса не выводить, пока поиск не реализован.
- Поддерживать читаемость: основной текст около 15–16 px.
- Не перегружать декоративной «тетрадностью».
- Карточки должны выглядеть нажимаемыми.
- Между секциями нужен воздух.

## Что реализовано

- Astro-проект со страницами:
  - `/`
  - `/notes/`
  - `/notes/[slug]/`
  - `/checklists/`
  - `/checklists/[slug]/`
  - `/about/`
- Контент через Astro Content Collections:
  - `notes`
  - `checklists`
  - `cases`
- Черновики `draft: true` исключаются из production-списков.
- Сохранены директории `docs/`, `design/`, `content/`.

npm install
npm run dev

## Структура

- `src/content/` — коллекции и demo-контент.
- `src/pages/` — страницы MVP.
- `src/layouts/`, `src/components/`, `src/styles/` — общие части интерфейса.

## Ограничения MVP

- Нет БД/CMS/админки.
- Нет темной темы.
- Нет поиска и сложной интерактивности.
